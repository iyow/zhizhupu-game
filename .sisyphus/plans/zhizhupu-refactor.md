## 一、代码架构重构

### 1.1 模块化拆分

将单文件 `game.js` 拆分为多个模块：

```
src/
├── game.js           # 主入口，初始化、事件绑定
├── state.js          # 游戏状态管理 (G对象)
├── dialog.js         # 对话系统
├── scene.js          # 场景切换、热区管理
├── puzzle.js         # 谜题系统
├── audio.js          # 音效系统
├── ui.js             # UI组件（通知、对话框等）
├── data/
│   ├── dialogs.js    # 对话数据
│   ├── hotspots.js   # 热区配置
│   └── puzzles.js    # 谜题配置
└── utils.js          # 工具函数
```

### 1.2 修复现有Bug

| Bug | 问题描述 | 修复方案 |
|-----|---------|---------|
| 热区系统 | `switchSceneWithHotspots` 定义但未使用 | 统一使用包装后的 `switchScene` |
| 重复绑定 | 热区可能重复创建 | 切换场景前清理旧热区 |
| 对话跳过 | 打字中点击行为不一致 | 统一处理逻辑 |
| 全局污染 | 大量全局函数 | 使用IIFE或ES6模块 |

## 九、注意事项

1. **保持向后兼容**: 存档格式不变
2. **性能优先**: 避免过多动画影响性能
3. **移动端适配**: 响应式设计
4. **可访问性**: 添加必要的ARIA标签
5. **保留现有内容**: 所有对话、剧情、谜题逻辑不变
